--- layout: page title: mfind subtitle: "Unix find(1) reimplementation in C — recursive, filtered, optionally parallel." ---

mfind is a simplified reimplementation of the Unix find utility written in C. It supports recursive directory traversal with filtering by name, type, size, and depth. When multiple start directories are provided, traversal can be parallelised using POSIX threads.

Memory safety was a primary concern throughout: the entire codebase has been verified with Valgrind (zero leaks, zero errors) under all supported flag combinations.

~/usage 

mfind [options] [path ...]

Options:
  -name <pattern>   filter by filename glob pattern
  -type f|d         filter by type: f=file, d=directory
  -size [+|-]N      filter by size in bytes (+ larger, - smaller)
  -maxdepth N       limit traversal depth
  -mindepth N       skip entries shallower than N
  -parallel         use one thread per start directory

~/build 

git clone https://github.com/{{ site.github_username }}/mfind
cd mfind
make
./mfind -name "*.c" -type f /usr/src
